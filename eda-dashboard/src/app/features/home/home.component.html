Comparative Analysis of Event-Driven Architectures

This dashboard visualizes benchmarking results for two implementations of the same order-processing workflow: .NET (MediatR + Kafka) versus Go (Kafka).

Stacks: 2
Scenarios: baseline • high-load • fault
Metrics: latency • throughput • cpu • memory
Workflow
Order Processing Pipeline
A standardized, end-to-end business flow implemented identically in both stacks, ensuring a fair comparison of event-driven execution.
Latency
P95 / P99
Tail-latency measurements across the full pipeline, capturing real-world behavior under load, including queueing and processing delays.
Throughput
Requests per Second
Sustained throughput under controlled test scenarios, used to identify bottlenecks related to runtime, IO, and message handling strategy.
Resources
CPU / Memory
Runtime overhead profiling to evaluate efficiency and scalability characteristics of .NET and Go under comparable workload intensity.
Fault Tolerance
Recovery & Resilience
Behavior under failure conditions such as network delays, consumer restarts, and broker unavailability, with emphasis on recovery time and stability.
